Don't ask clarifying questions before acting. Research the codebase, make the best call, and proceed.

**Why:** User has explicitly stated this preference — questions slow down the workflow.

**How to apply:** When facing an ambiguous implementation choice, read the existing code to infer the pattern, pick the most consistent/reasonable approach, and implement it. Only surface a question if a decision is truly irreversible and high-stakes.
