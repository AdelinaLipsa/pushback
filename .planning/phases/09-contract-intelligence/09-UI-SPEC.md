## Spacing Scale

Declared values (multiples of 4 throughout the project):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap: '0.25rem'), badge padding |
| sm | 8px | Inline element spacing, label-to-input gap |
| md | 16px | Default padding within cards, form field spacing |
| lg | 20px | Card header bottom margin (1.25rem), section internal gap |
| xl | 24px | Card padding (1.5rem), section padding |
| 2xl | 32px | Major section breaks |
| 3xl | 48px | Page-level spacing |

Exceptions:
- "What to say back" container padding: 14px (0.875rem) — matches existing ClauseCard pattern
- Risk indicator dot: 8px diameter (0.5rem width/height)
- Touch targets: minimum 44px height for all interactive buttons

Source: existing ClauseCard.tsx, SituationPanel.tsx, ResponseOutput.tsx spacing patterns.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 | 1.6 |

Phase-specific typography rules:
- "Based on your contract:" label: 13px (0.8rem), weight 400, color var(--text-muted) (#52525b), uppercase, letter-spacing 0.07em — matches the existing "What to say back" label pattern in ClauseCard
- Clause titles in attribution list: 14px (0.875rem), weight 400, color #52525b (text-[#52525b]) — comma-separated inline, no bullets

Source: ClauseCard.tsx, SituationPanel.tsx, ResponseOutput.tsx type patterns.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0a0a0a (var(--bg-base)) | Page background |
| Secondary (30%) | #111111 / #1a1a1a (var(--bg-surface) / var(--bg-elevated)) | Cards, panels, elevated containers |
| Accent (10%) | #84cc16 (var(--brand-lime)) | Primary CTAs, focus rings, icon accents, lime indicator dot (low/medium risk) |
| High-risk indicator | #f59e0b (raw hex — not a CSS variable) | Risk indicator dot for high risk level only |
| Critical / Destructive | #ef4444 (var(--urgency-high)) | Risk indicator dot for critical risk level; destructive actions |

Accent (lime) reserved for:
- Primary action buttons (btnStyles.primary)
- Input focus border (onFocus borderColor)
- Sparkles icon in ResponseOutput header
- Risk indicator dot for low and medium risk levels

Risk indicator dot color mapping (D-13):
- `low` → #84cc16 (var(--brand-lime))
- `medium` → #84cc16 (var(--brand-lime))
- `high` → #f59e0b (raw hex, matches --brand-amber in CONTEXT.md; NOTE: --brand-amber in globals.css is mapped to lime #84cc16 — use raw hex #f59e0b for high-risk dots, not the CSS variable)
- `critical` → #ef4444 (var(--urgency-high))

Risk indicator text color: always var(--text-secondary) (#a1a1aa), never changes with risk level (D-13).

Source: globals.css CSS variables, 09-CONTEXT.md D-13, lib/ui.ts RISK_COLORS_RICH.

## Component Specifications

### ResponseOutput — "Based on your contract:" Section (D-01, D-02, D-03, D-04)

**Render condition:** Only when `contractClausesUsed` prop is a non-empty string array. Hidden when array is empty or prop is absent.

**Placement:** Below the response text box (`var(--bg-base)` pre block), above the action buttons row.

**Visual structure:**
```
[micro-label: "BASED ON YOUR CONTRACT:"]
[comma-separated clause titles in text-[#52525b]]
```

**Styling:**
- Container: no additional background card — render inline within the existing ResponseOutput card, separated by `marginTop: '0.75rem'` from the text box
- Label: uppercase, fontSize 11px (0.7rem), letterSpacing 0.07em, color var(--text-muted) (#52525b), marginBottom 4px (0.25rem)
- Clause title list: fontSize 13px (0.8rem), color #52525b, lineHeight 1.5, comma-separated inline text — no bullets, no badges, no icons
- Example rendered: "Revision clause, Payment terms, IP ownership"

**New prop:** `contractClausesUsed?: string[]` added to ResponseOutputProps

Source: 09-CONTEXT.md D-01 through D-04; discretion note on exact formatting applies.

### SituationPanel — Risk-Aware Indicator (D-11, D-12, D-13, D-14)

**Render condition:** Only when `hasContract` prop is `true`. Hidden when `hasContract` is false or absent.

**Placement:** In the SituationPanel header row, below the tool name/close button row, as a secondary row element — or appended inline after the tool label if horizontal space permits. Recommended: separate row below the header with `marginTop: '0.5rem'` and `marginBottom: '-0.5rem'` (tight to form).

**Visual structure:**
```
[●] ["{RiskLevel} risk contract loaded"]
```
- Dot: 8px × 8px circle (borderRadius '50%', display 'inline-block'), flexShrink 0
- Text: "{RiskLevel} risk contract loaded" where RiskLevel is capitalized (e.g. "High risk contract loaded", "Low risk contract loaded")
- Layout: flex row, alignItems center, gap 6px (0.375rem)

**Dot color by risk level (D-13):**
- low → backgroundColor: '#84cc16' (var(--brand-lime))
- medium → backgroundColor: '#84cc16' (var(--brand-lime))
- high → backgroundColor: '#f59e0b'
- critical → backgroundColor: '#ef4444' (var(--urgency-high))

**Text color:** always var(--text-secondary) (#a1a1aa), fontSize 13px (0.8rem), weight 400

**New props added to SituationPanel:**
- `hasContract?: boolean`
- `contractRiskLevel?: RiskLevel`

Source: 09-CONTEXT.md D-11 through D-14.

### ClauseCard — Copy Button on "What to say back" (D-15, D-16)

**Placement:** Inline with the "What to say back" label — label and button on the same flex row (justifyContent: 'space-between').

**Button appearance:** Ghost icon button — no background, no border, using the Lucide `Copy` icon at 14px (size={14}), color var(--text-muted) (#52525b). On hover: color var(--text-secondary). After copy: icon switches to Lucide `Check` at 14px, color var(--brand-lime), auto-resets after 2000ms.

**Copy behavior (D-16):** `navigator.clipboard.writeText(clause.pushback_language)` — no DB tracking, no responseId.

**Do not reuse the full `CopyButton` component** (it is sized for a primary action with large padding and full "Copy Message" label text). Write a minimal inline copy icon button within ClauseCard directly.

**Label row layout:**
```jsx
<div style={{ display: 'flex', alignItems: 'center', justifyContent: 'space-between', marginBottom: '0.4rem' }}>
  <p style={{ color: 'var(--text-muted)', fontSize: '0.7rem', textTransform: 'uppercase', letterSpacing: '0.07em', margin: 0 }}>What to say back</p>
  <button ...> {/* Lucide Copy icon, 14px, ghost */ }
  </button>
</div>
```

Source: 09-CONTEXT.md D-15, D-16; ClauseCard.tsx existing pattern; CopyButton.tsx comparison.
