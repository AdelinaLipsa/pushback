## Spacing Scale

Declared values (must be multiples of 4). Source: inferred from existing auth pages (inline styles converted to logical tokens).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, divider height (1px separator), inline label margin |
| sm | 8px | Label-to-input gap (`marginBottom: '0.5rem'`), paragraph spacing in prose |
| md | 16px | Default element spacing, gap between prose sections |
| lg | 24px | Section padding, card padding (`padding: '2rem'` = 32px — use 24px for prose sections) |
| xl | 32px | Card padding (`padding: '2rem'`), major section breaks in policy content |
| 2xl | 48px | Page top/bottom padding, separation between major policy sections |
| 3xl | 64px | Page-level vertical padding on full-height wrapper |

Exceptions:
- Legal page container `maxWidth: 680px` (wider than auth 400px to accommodate prose columns)
- Legal page card `padding: 2.5rem` (40px) on desktop — matches content density expectations for policy prose
- Link touch targets: minimum 44px height for `/terms` and `/privacy` inline links in signup footer

## Interaction States

### /privacy and /terms Pages

| State | Behavior |
|-------|----------|
| Default | Static prose page — no loading, no interaction required |
| Contact email link | `color: var(--brand-amber)` on hover/focus — no underline at rest, underline on hover |
| Back link | `color: var(--brand-amber)` — hover: `opacity: 0.8` |
| Page scroll | No sticky header; full-page scroll |

### Signup Footer Links

| State | Behavior |
|-------|----------|
| Default | `color: var(--brand-amber)`, `fontWeight: 500`, `textDecoration: 'none'` |
| Hover | `textDecoration: 'underline'` — consistent with auth page link pattern |
| Focus (keyboard) | Browser default focus outline via `outline-ring/50` (globals.css `@layer base`) |

### Email Templates

| State | Behavior |
|-------|----------|
| CTA button (email) | Static inline styles only — no hover state possible in email HTML |
| Fallback rendering | Plain text fallback content is not required for Resend-delivered HTML emails; Resend handles text/plain generation |
