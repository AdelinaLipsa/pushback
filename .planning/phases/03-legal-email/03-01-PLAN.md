## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous browser → public page | Anyone can GET /privacy and /terms without auth — intentional |
| Anonymous browser → /signup | Existing boundary; link update does not change trust model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Legal page content | mitigate | Content is static hardcoded JSX in Server Components — no user input is rendered into the page, so XSS via prose content is not possible. No `dangerouslySetInnerHTML`. |
| T-03-02 | Information Disclosure | Contact email in legal pages | accept | `adelina.lipsa@gmail.com` is intentionally published as the contact address per D-01. Risk of scraping accepted. |
| T-03-03 | Spoofing | Link target manipulation | mitigate | `<Link href="/terms">` and `<Link href="/privacy">` use absolute internal paths — cannot be redirected to external phishing sites. |
| T-03-04 | Denial of Service | Unauthenticated page load | accept | Static Server Component with no DB or external calls — no amplification surface. Standard Next.js rendering. |
| T-03-05 | Information Disclosure | Privacy Policy completeness | mitigate | Policy explicitly names every third-party processor (Anthropic, Supabase, Vercel, Stripe, Resend) — required for GDPR Art. 13 transparency. |
</threat_model>
