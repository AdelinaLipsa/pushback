<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App server → Resend API | Outbound transactional email; RESEND_API_KEY crosses this boundary. Key is server-only (never exposed to client). |
| App server → user's email inbox | Email content is delivered to the recipient; content is crafted from app-controlled templates + server-provided parameters only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information Disclosure | `RESEND_API_KEY` env var | mitigate | Key read server-side only in `lib/email.ts`; module has no `'use client'` marker and is imported only by route handlers. Key never appears in bundler client output (no `NEXT_PUBLIC_` prefix). |
| T-03-07 | Tampering | Email HTML content | mitigate | Welcome template is fully static (no user-supplied interpolation). Upgrade template interpolates ONLY `billing.amount` and `billing.nextBillingDate` — both derived server-side from Stripe webhook data (Plan 04 formats them). No user-supplied strings reach the HTML. |
| T-03-08 | Spoofing | Sender address | mitigate | Sender comes from `RESEND_FROM_EMAIL` env var, which MUST be an address on a Resend-verified domain. SPF/DKIM/DMARC handled at the domain-verification level, not in code. |
| T-03-09 | Information Disclosure | Resend error details leaked to clients | mitigate | `sendXxxEmail` throws a generic `Error` with Resend's message. Callers (Plans 03/04) wrap in `.catch()` and `console.error` only — they do not echo the error into HTTP responses. |
| T-03-10 | Denial of Service | Resend rate limits / outage | accept | Email is fire-and-forget at call sites. A Resend outage logs an error but does not block the user's auth callback redirect or webhook 200 response. Accepted risk for v1. |
| T-03-11 | Repudiation | No email send audit trail | accept | For v1, `console.error` on failure is sufficient; Sentry/structured logging comes in Phase 5 (OBS-01). Accepted risk. |
</threat_model>
