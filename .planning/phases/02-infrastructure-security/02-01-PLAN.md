<objective>
Fix two webhook reliability bugs: add a cookie-free Supabase admin client for use in webhook context (INFRA-01), and add a loud fail-fast guard when CREEM_WEBHOOK_SECRET is missing (INFRA-02).

Purpose: The current webhook uses createServiceSupabaseClient() which calls await cookies() — this throws "cookies() was called outside a request scope" in webhook handlers that run without a Next.js request context. Additionally, if CREEM_WEBHOOK_SECRET is unset, the HMAC check silently fails and paid upgrades are silently dropped.

Output:
- lib/supabase/server.ts gains createAdminSupabaseClient() — synchronous, no cookies, bare createClient
- app/api/webhooks/creem/route.ts gains a secret guard before body read, and switches to createAdminSupabaseClient()
</objective>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Missing CREEM_WEBHOOK_SECRET silently allows all webhook events through (HMAC check computes with undefined → mismatch always, real events dropped without alert) | HIGH | Guard returns 500 before body read — loud failure visible in logs/monitoring on every request |
| cookies() called in webhook scope throws unhandled runtime error, crashing the handler silently in some environments | HIGH | Switch to createAdminSupabaseClient() which uses bare createClient with no cookie adapter |
| Service role key logged or exposed in error responses | MED | createAdminSupabaseClient() uses env var with ! assertion — never interpolated into response strings |
| SUPABASE_SERVICE_ROLE_KEY unset causes panic at createAdminSupabaseClient call site | MED | Accept — env var absence causes TypeScript non-null assertion to throw, which is acceptable fail-loud behavior for misconfiguration; mirrors existing pattern for NEXT_PUBLIC_SUPABASE_URL! |
