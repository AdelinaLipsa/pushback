- script-src 'self' 'unsafe-inline': required for Next.js 16 RSC inline script tags (__next_f.push)

- font-src 'self': next/font/google self-hosts fonts at /_next/static/media/ — no external font CDN needed
- connect-src 'self' https://*.supabase.co wss://*.supabase.co: covers all Supabase subdomains including auth, db, storage, realtime WebSocket
- object-src 'none': blocks Flash/plugin exploitation
- base-uri 'self': prevents base tag injection
- form-action 'self': Next.js Server Actions POST to same origin
- frame-ancestors 'none': clickjacking prevention (alongside X-Frame-Options: DENY for older browsers)
- upgrade-insecure-requests: forces HTTPS for mixed-content resources
- NO unsafe-eval: not needed by this stack (per D-06 constraint)

## Threats

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Clickjacking — attacker embeds app in iframe to capture clicks on UI elements | HIGH | X-Frame-Options: DENY + frame-ancestors 'none' in CSP (both for maximum browser compat) |
| Protocol downgrade — HTTP request strips secure cookie flags, exposes session tokens | HIGH | Strict-Transport-Security: max-age=63072000; includeSubDomains; preload (2-year HSTS with preload eligibility) |
| Tech stack fingerprinting via X-Powered-By: Next.js header | MED | poweredByHeader: false removes this header globally |
| Cross-site referrer leaking sensitive URL paths in Referer header | MED | Referrer-Policy: strict-origin-when-cross-origin — sends origin only on cross-origin, full URL on same-origin |
| XSS via injected inline scripts | MED | CSP script-src 'self' 'unsafe-inline' — partial mitigation; unsafe-inline is required for Next.js hydration so cannot be removed; frame-ancestors and object-src block escalation paths |
| Base tag injection redirecting relative URLs to attacker origin | MED | base-uri 'self' — prevents base tag manipulation |
| CSP blocks Next.js hydration causing blank pages in production | HIGH | unsafe-inline present in script-src and style-src — verified correct for Next.js 16 non-nonce CSP path |
</threat_model>

```typescript
import type { NextConfig } from 'next'

const cspHeader = `
  default-src 'self';
  script-src 'self' 'unsafe-inline';
  style-src 'self' 'unsafe-inline';
  img-src 'self' blob: data:;
  font-src 'self';
  connect-src 'self' https://*.supabase.co wss://*.supabase.co;
  object-src 'none';
  base-uri 'self';
  form-action 'self';
  frame-ancestors 'none';
  upgrade-insecure-requests;
`

const nextConfig: NextConfig = {
  poweredByHeader: false,
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'Content-Security-Policy',
            value: cspHeader.replace(/\n/g, ''),
          },
          {
            key: 'Strict-Transport-Security',
            value: 'max-age=63072000; includeSubDomains; preload',
          },
          {
            key: 'X-Frame-Options',
            value: 'DENY',
          },
          {
            key: 'Referrer-Policy',
            value: 'strict-origin-when-cross-origin',
          },
        ],
      },
    ]
  },
}

export default nextConfig
```
