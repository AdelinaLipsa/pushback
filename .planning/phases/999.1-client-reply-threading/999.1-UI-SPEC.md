## Interaction Contracts

### 1. "Paste their reply" trigger button

- Location: Below the existing `CopyButton` in the expanded response card body (after the pre block).
- Style: `btnCls.ghost` from `lib/ui.ts` — `background: none; border: none; color: var(--brand-lime); font-size: 0.9rem; cursor: pointer; padding: 0.5rem 0`.
- Icon: `MessageSquarePlus` (16px, lime, `aria-hidden`), label: "Paste their reply →"
- Behavior: Clicking toggles a `replyOpen` boolean on the response card. Once a reply thread exists for this response (loaded from joined data), this button is permanently replaced by the `ReplyThreadCard` — it does not toggle back.
- Accessibility: `aria-expanded={replyOpen}`, `aria-controls="reply-textarea-{responseId}"`.

### 2. Reply textarea (collapsed → expanded)

- Appears when `replyOpen === true` and no existing `reply_threads` row for this response.
- Expand animation: `fadeUp` CSS animation (0.45s, defined in globals.css) — matches the pattern used in DefenseDashboard analyze card.
- Textarea style: matches `inputStyle` from `lib/ui.ts` — `var(--bg-base)` background, `var(--bg-border)` border, `var(--text-primary)` text, `0.9rem`, `border-radius: 0.5rem`, `padding: 0.75rem`.
- Dimensions: `width: 100%`, `min-height: 120px`, `resize: vertical`.
- Placeholder: "Paste what they replied..."
- Character limit indicator: shown below textarea when count > 4500 characters, format: "{n}/5000" in `var(--text-muted)` at `0.75rem`.
- No label element above the textarea (placeholder is sufficient given the button label context).

### 3. "Analyze reply" submit button

- Style: `btnCls.primary` from `lib/ui.ts` — lime background, #0a0a0a text, 600 weight, `border-radius: 0.5rem`.
- Label: "Analyze reply →"
- Loading state label: "Analyzing…" with `disabled` + `opacity: 0.7`.
- The button is disabled when `client_reply.trim().length < 10` (Zod min enforced client-side too).
- Positioned to the right of a "Cancel" ghost button that collapses the textarea (`replyOpen = false`).

### 4. ReplyThreadCard (post-analysis display)

This component replaces the textarea after analysis completes, and is also shown on initial page load when a reply thread already exists in the joined data.

Structure (top-to-bottom, all within expanded card body):

```
[borderLeft accent strip — 3px, stance color]
[Signal row: Icon (16px, stance color) + Label (stance color, 0.9rem, weight 600)]
[Explanation line: 0.75rem, var(--text-muted), 1.5 line-height]
[Divider: 1px solid var(--bg-border), margin 12px 0]
[Follow-up pre block: same style as response card pre block]
[CopyButton: reuse existing component, label="Copy Follow-Up"]
```

Container style:
```
backgroundColor: 'var(--bg-surface)'
border: '1px solid var(--bg-border)'
borderLeft: '3px solid {stanceColor}'
borderRadius: '0.5rem'
padding: '1rem 1.25rem'
marginTop: '0.75rem'
```

Icon-to-label gap: 6px (not in standard scale — exception, matches existing app icon+label patterns).

### 5. Stance icon + label mapping

| Signal | Lucide Icon | Label | Color |
|--------|-------------|-------|-------|
| `backing_down` | `TrendingDown` | "Backing down" | `var(--brand-lime)` |
| `doubling_down` | `TrendingUp` | "Doubling down" | `var(--urgency-medium)` |
| `escalating` | `AlertTriangle` | "Escalating" | `var(--urgency-high)` |
| `unclear` | `HelpCircle` | "Unclear" | `var(--urgency-medium)` |

Note: `TrendingDown` and `TrendingUp` are already imported in `DefenseDashboard.tsx`. `AlertTriangle` is already imported. `HelpCircle` must be added to imports at the call site.

All icons: `size={16}`, `aria-hidden={true}`. The label text carries the semantic meaning for screen readers.

### 6. Loading / error states

- Loading: Button shows "Analyzing…", disabled. No spinner component — matches existing pattern (analyze-message uses text-only loading state).
- API error: Shown as inline text below the submit button in `var(--urgency-high)`, `0.8rem`, `role="alert"`. Error message content follows error copy contract below.
- Upgrade required (403): Shown as inline text: "You've used all your credits. Upgrade to Pro to continue." with a `btnCls.primary` "Upgrade to Pro" button below it (same `handleUpgrade` / `startCheckout` pattern used throughout the app).

### 7. Already-analyzed state (on page load)

When the history page joins `reply_threads` data and a reply thread exists for a response, the ReplyThreadCard is rendered immediately inside the expanded card body — no "Paste their reply" button shown, no textarea. The response card still shows its existing CopyButton above the ReplyThreadCard.
