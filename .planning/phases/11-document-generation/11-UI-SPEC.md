## Spacing Scale

| Token | Value | Usage |

| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing — e.g. gap between document action buttons |
| md | 16px | Default element spacing — e.g. DocumentOutput internal padding |
| lg | 24px | Section padding — e.g. DocumentOutput card padding (p-6 = 1.5rem) |

## Component Inventory

### New Components

#### `components/defense/DocumentOutput.tsx`

Layout: card (`bg-bg-surface border border-bg-border rounded-2xl p-6`) with `response-enter` animation class on mount.

Internal structure (top to bottom):
1. **Header row** — `flex items-center justify-between mb-5`
   - Left: document type label (e.g. "SOW Amendment") at `font-semibold text-sm`
   - Right: Back button — `← Back to message` as ghost-style button, calls `onBack()` prop
2. **Pre block** — `bg-bg-base border border-bg-border rounded-lg p-5 mb-4`
   - `<pre className="font-mono text-sm leading-relaxed text-text-primary whitespace-pre-wrap break-words m-0">`
3. **Edit note** — always visible below pre block, above copy button:
   - `text-text-secondary text-xs leading-relaxed`
   - See Copywriting Contract for exact text
4. **Copy button row** — `flex items-center gap-3 mt-4`
   - Reuse `CopyButton` component with `text={document}` prop — no `responseId` (documents are ephemeral, no DB row)

Props interface:
```
interface DocumentOutputProps {
  document: string
  documentType: 'sow_amendment' | 'kill_fee_invoice' | 'dispute_package'
  onBack: () => void
}
```

#### Document Button Row (modification to `ResponseOutput.tsx`)

Add a secondary row below the existing `flex items-center gap-3 mt-5` action row. The new row is:
- `mt-4 pt-4 border-t border-bg-border` — visually separates from primary Copy + Mark as sent row
- Contains 1 button (type-specific, see Copywriting Contract)
- Button style: `btnCls.outline` from `lib/ui.ts` — `bg-transparent border border-bg-border text-text-muted text-sm rounded-lg`
- Loading state: replace button text with "Generating document…" + disable button; apply `lime-pulse-border` class to button while `documentGenerating` is true
- Only renders when `onGenerateDocument` prop is provided (i.e. always, since D-05 says button is always visible post-response)

New props on `ResponseOutput`:
```
onGenerateDocument: (type: DocumentType) => void
documentGenerating?: boolean
```

### State additions to `DefenseDashboard.tsx`

Three new state fields (D-13):
- `documentLoading: boolean` — initialized `false`
- `documentOutput: string | null` — initialized `null`
- `documentError: string | null` — initialized `null`

Render logic (D-03 / D-04): when `documentOutput !== null`, render `DocumentOutput` instead of `ResponseOutput` in the right panel. `response` state is NOT cleared — back button restores ResponseOutput view.

## Interaction Contracts

### Document Generation Flow

```
ResponseOutput rendered (response exists)
  └─ User clicks document button (ghost/outline style, below primary row)
       ├─ plan === 'free' → setShowUpgrade(true) (UpgradePrompt shown, same pattern as handleGenerate)
       └─ plan === 'pro'  → handleGenerateDocument(type) called
            ├─ setDocumentLoading(true)
            ├─ setDocumentError(null)
            ├─ POST /api/projects/[id]/document { document_type, context }
            │    ├─ success → setDocumentOutput(result.document), setDocumentLoading(false)
            │    └─ error  → setDocumentError("Document generation failed — please try again."), setDocumentLoading(false)
            └─ DocumentOutput renders (replaces ResponseOutput in right panel)
```

### Back Navigation

```
DocumentOutput → user clicks "← Back to message"
  └─ onBack() called → setDocumentOutput(null) in DefenseDashboard
       └─ response state untouched → ResponseOutput re-renders with original message
```

### Error Display

`documentError` displays **inline**, not as a toast. Location: below the document button row inside ResponseOutput (same position as the button, replacing it when error is set). Style: `text-xs text-urgency-high mt-2`. Rationale: toast would require a toast library not yet present; inline error is consistent with how `analyzeError` is displayed in the existing SituationPanel analyze card (D-13 reference in STATE.md).

## Accessibility

- Back button: `aria-label="Back to generated message"` on the button element
- Document `<pre>` block: `role="region"` + `aria-label="Generated document"` on the containing div
- Document generate buttons: `disabled` attribute set + `aria-busy="true"` while `documentGenerating` is true
- Error message: `role="alert"` on the inline error `<span>` so screen readers announce it immediately
