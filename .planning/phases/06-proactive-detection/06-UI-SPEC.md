## State Contract

New state variables in DefenseDashboard (from D-09, locked):

| Variable | Type | Initial Value | Description |
|----------|------|---------------|-------------|
| `messageInput` | `string` | `''` | Controlled value of the message textarea |
| `analyzeLoading` | `boolean` | `false` | True while POST /analyze-message is in-flight |
| `analysisResult` | `{ tool_type: DefenseTool; explanation: string; situation_context: string } \| null` | `null` | Populated on successful classification |

Interaction sequence:
1. User types in textarea → `messageInput` updates
2. User clicks "Analyze Message" → `analyzeLoading = true`, POST `/api/projects/[id]/analyze-message`
3. On success → `analyzeLoading = false`, `analysisResult = { ... }`, `selectedTool = DEFENSE_TOOLS.find(t => t.type === analysisResult.tool_type)`, result banner renders, SituationPanel renders with `initialSituation={analysisResult.situation_context}`
4. On 403 UPGRADE_REQUIRED → `analyzeLoading = false`, `showUpgrade = true` (same as generate path)
5. On other error → `analyzeLoading = false`, `analyzeError` renders (see Error State below)
6. User clicks "Start over" → `analysisResult = null`, `messageInput = ''`, `selectedTool = null`, `response = null`

## Interaction States

### Analyze Section — Idle
- Textarea empty, button disabled (opacity 0.6)
- No result banner visible

### Analyze Section — Has Input
- Textarea has text, button enabled (full opacity)
- Character count visible below textarea

### Analyze Section — Loading
- Button label changes to "Analyzing...", button disabled
- Textarea remains enabled (user can read their input)
- No spinner needed — button copy alone is sufficient (consistent with SituationPanel "Writing your message..." pattern)

### Analyze Section — Result Shown
- Result banner slides in with `response-enter` animation
- Analyze section card remains visible above (shows the original input)
- SituationPanel appears below result banner, pre-filled
- "Start over" link appears below result banner

### Analyze Section — Error
- Error message renders below the Analyze button
- Style: `color: 'var(--urgency-high)'`, `fontSize: '0.85rem'`, `marginTop: '0.5rem'`
- Matches `generateError` display pattern in existing DefenseDashboard (lines 134–136)

### Plan-Gated State (isAtLimit)
- Analyze button replaced by UpgradePrompt trigger (same `setShowUpgrade(true)` path as tool selection at limit)
- Button label when at limit: "Analyze Message" button is hidden; instead, `selectTool`-style gate fires `setShowUpgrade(true)` on click
- Alternative: disable the Analyze button and show inline copy "Upgrade to analyze messages" — executor chooses simpler of the two
