## Typography

Inherits project-wide Inter font. Phase 12 uses exactly 3 type sizes.

| Role | Size | Weight | Line Height | Usage in Phase 12 |
|------|------|--------|-------------|-------------------|
| Label | 11.2px (0.7rem) | 600 | 1.2 | Badge pill text (matches existing status / contract risk pill size) |
| Body | 14px (0.875rem) | 400 | 1.5 | Signal text in ClientBehaviorCard signal list; score range label (color: var(--text-muted) distinguishes it from primary text at the same size); badge row secondary text |
| Card heading | 14px (0.875rem) | 600 | 1.4 | ClientBehaviorCard section title ("Client Risk") — weight 600 distinguishes heading from body at the same size |
| Score display | 24px (1.5rem) | 600 | 1.0 | Numeric score inside ClientBehaviorCard — size alone carries the hierarchy |

Size rationale: 0.7rem, 0.875rem, and 1.5rem form three perceptually distinct steps. Within the 0.875rem tier, weight (400 vs 600) and color (var(--text-secondary) vs var(--text-muted)) handle all sub-role distinctions. No fourth size is needed.

Font weights used: 400 (regular) and 600 (semibold) only.

### Risk Level Colors (new constant: CLIENT_RISK_COLORS)

Three colors reserved exclusively for client behavioral risk rendering. These are distinct from the existing RISK_COLORS (which map contract risk: low/medium/high) to prevent semantic confusion.

| Level | Score Range | Color Hex | CSS Var (existing) | Usage |
|-------|------------|-----------|-------------------|-------|
| green | 0–25 | `#22c55e` | `var(--brand-green)` | Badge border + text; ClientBehaviorCard left accent |
| yellow | 26–60 | `#f97316` | `var(--urgency-medium)` | Badge border + text; ClientBehaviorCard left accent |
| red | 61–100 | `#ef4444` | `var(--urgency-high)` | Badge border + text; ClientBehaviorCard left accent |

Rationale: Reusing `var(--urgency-medium)` for yellow keeps the warning semantic consistent with how the project already uses that orange for urgency-medium tools. No new tokens needed.

Accent (`var(--brand-lime)`) is NOT used for risk levels. Lime is exclusively for: primary CTAs, "Handle now →" links, lime-pulse border animations, and brand highlight moments.

## Component Specifications

### 1. ClientRiskBadge (new pill in ProjectCard badge row)

**File:** `components/project/ClientRiskBadge.tsx` (new)
**Placement:** Inside the existing badge row flex container in `ProjectCard.tsx`, after the contract risk badge and before or after OVERDUE pill.
**Render condition:** Always rendered (D-02: green badge at 0 is a positive signal).

**Visual anatomy:**
- Pill shape: `borderRadius: '9999px'`
- Background: `rgba(0,0,0,0.3)` — matches contract risk badge background exactly (D-04)
- Border: `1px solid {CLIENT_RISK_COLORS[level]}` — level-matched color
- Text color: `{CLIENT_RISK_COLORS[level]}` — matches border
- Font size: 0.7rem (11.2px) — matches existing badge pills in the row
- Font weight: 600
- Padding: `0.25rem 0.5rem` — matches existing badge pills (4px / 8px on the 8-point scale)
- Letter spacing: none (no uppercase, unlike status pill)

**Label format:** `Client {score}` — e.g. "Client 0", "Client 40", "Client 72"
Rationale: "Client" prefix distinguishes this from the contract badge ("Risk 7/10"). The word makes clear this measures a person's behavior, not a document's terms.

**Props:**
```
score: number      // 0–100
level: 'green' | 'yellow' | 'red'
```

### 2. ClientBehaviorCard (new panel on project detail page)

**File:** `components/project/ClientBehaviorCard.tsx` (new)
**Placement:** Project detail page (`app/(dashboard)/projects/[id]/page.tsx`), rendered above `DefenseDashboard` (D-11), below `ProjectHeader`.
**Render condition:** Only when `score > 0` (D-10). Zero-activity projects get no card.

**Visual anatomy:**
- Outer container: `backgroundColor: 'var(--bg-surface)'`, `border: '1px solid var(--bg-border)'`, `borderLeft: '4px solid {CLIENT_RISK_COLORS[level]}'`, `borderRadius: '0.875rem'`, `padding: '1rem 1.5rem'`
- Left accent border width: 4px — matches the `ProjectCard` hover lime border width for visual consistency
- No filled background tint — the left accent strip carries the severity signal

**Header row (flex, space-between):**
- Left: Section label "Client Risk" in 0.875rem / weight 600 / `var(--text-secondary)`
- Right: Score display — `{score}` in 1.5rem / weight 600 / `{CLIENT_RISK_COLORS[level]}`; followed by score range label in 0.875rem / weight 400 / `var(--text-muted)` — e.g. "72 · High risk" or "40 · Watch this client" or "12 · No concerns"

**Score range labels:**
| Level | Score | Label |
|-------|-------|-------|
| green | 0–25 | "No concerns" |
| yellow | 26–60 | "Watch this client" |
| red | 61+ | "High-risk client" |

**Signal list (below header row, marginTop: 0.5rem):**
- Each signal: one row, flex, gap 8px
- Signal icon: Lucide icon, 14px, `var(--text-muted)`, `flexShrink: 0`
- Signal text: 0.875rem / weight 400 / `var(--text-secondary)`
- No bullet points, no list markers — icon-text rows only
- Vertical gap between signals: 8px (`gap: '0.5rem'`)
- Empty signal list: card does not render (score = 0 gate covers this)

**Signal icon assignments:**
| Signal category | Lucide Icon |
|-----------------|-------------|
| Scope-related (scope_change, moving_goalposts) | `Layers` |
| Payment-related (payment overdue, chargeback_threat, retroactive_discount) | `CreditCard` |
| Dispute / threat (dispute_response, review_threat) | `ShieldAlert` |
| Ghost / communication (ghost_client, feedback_stall) | `EyeOff` |
| Contract / IP (ip_dispute, kill_fee, post_handoff_request) | `FileText` |
| Financial pressure (discount_pressure, rate_increase_pushback, rush_fee_demand, disputed_hours) | `TrendingDown` |
| Delivery (revision_limit, spec_work_pressure) | `RefreshCw` |

**Signal label format (human-readable, D-12):**
- Scope: "3 scope changes", "1 approved-then-rejected"
- Payment: "Payment overdue 12 days", "1 chargeback threat", "1 post-delivery discount demand"
- Dispute: "1 formal dispute", "1 review threat"
- Ghost: "1 ghost client situation", "2 feedback delays"
- Contract: "1 IP dispute", "1 kill fee invoked", "1 post-handoff request"
- Financial: "2 rate negotiations", "1 disputed hours", "1 rush-without-fee demand"
- Delivery: "3 revision limit situations"
- Overdue flag: "Payment currently overdue" (from isOverdue, not from sent responses)

Format rule: singular count uses plain form ("1 scope change"), plural uses count + plural noun ("3 scope changes"). Do not use "×" multiplier notation.

**Props:**
```
score: number           // 0–100
level: 'green' | 'yellow' | 'red'
signals: Array<{
  icon: string          // Lucide icon name
  label: string         // Human-readable signal text
}>
```

**Row anatomy (follows AttentionAlert pattern exactly):**

**CTA label:** "View project →"
**Spotlight rule (D-13):** Show the single highest-score project when multiple are yellow/red. One row per "highest risk client" insight, not one row per risky project.

## Scoring Contract (for computation clarity)

**Function signature:** `computeClientRisk(project: Project & { defense_responses?: DefenseResponse[] }): ClientRiskResult`
**Return type:**
```
{
  score: number           // 0–100, clamped
  level: 'green' | 'yellow' | 'red'
  signals: ClientRiskSignal[]
}
```

**Severity weights (D-06):**

| Tool type | Weight | Rationale |
|-----------|--------|-----------|
| `chargeback_threat` | 30 | Highest legal/financial exposure |
| `dispute_response` | 25 | Formal conflict escalation |
| `review_threat` | 20 | Reputational threat requiring response |
| `ip_dispute` | 20 | Legal complexity |
| `retroactive_discount` | 18 | Bad-faith post-delivery move |
| `moving_goalposts` | 15 | Approved-then-rejected is serious scope abuse |
| `kill_fee` | 15 | Mid-project cancellation conflict |
| `post_handoff_request` | 12 | Closed project reopened |
| `scope_change` | 10 | Per sent response — scope creep signal |
| `payment_final` | 10 | Reached the last-notice stage |
| `disputed_hours` | 10 | Active financial dispute |
| `payment_second` | 8 | Escalated to firm second reminder |
| `discount_pressure` | 8 | Rate negotiation attempt |
| `rush_fee_demand` | 6 | Timeline pressure |
| `rate_increase_pushback` | 6 | Client pushback on rate |
| `ghost_client` | 6 | Communication breakdown |
| `feedback_stall` | 4 | Delay causing timeline pressure |
| `revision_limit` | 4 | Exceeded revisions |
| `payment_first` | 3 | First payment reminder |
| `spec_work_pressure` | 3 | Spec/exposure request |
| `delivery_signoff` | 0 | Normal delivery — no risk signal |

Single chargeback_threat (weight 30) + overdue payment bonus = scores into yellow territory (26+) confirming D-06 calibration.

**Overdue payment contribution:** +15 to score when `payment_due_date` is past and `payment_received_at` is null. This is an independent additive contribution from the `isOverdue` field, not from sent responses.

**On-time payment bonus (D-08):** Include -10 when `payment_received_at` is not null AND `payment_received_at < payment_due_date`. Applies only once per project. Keeps the range clean for reliable clients.

**Score ceiling:** `Math.min(score, 100)`. Score floor: 0 (never negative).

**Filter:** Only `defense_responses` where `was_sent === true` contribute to score (D-05).

**No time decay (D-07):** All history counts permanently.

## Interaction Contract

### Hover / Focus States
- `ClientRiskBadge` has no hover state (it is non-interactive, inside a link card that already handles hover)
- `ClientBehaviorCard` has no hover state (display-only panel)
- Dashboard risk insight row: same hover behavior as existing `AttentionAlert` rows — none on the container; "View project →" link uses `hover:opacity-80 transition-opacity` (existing class)

### Animation
- `ClientBehaviorCard` uses `fade-up` animation class (existing keyframe in globals.css) with `animationDelay: '0.1s'` — consistent with how ProjectDetailClient renders cards
- Dashboard risk row uses `fade-up` with delay matching existing attention item pattern (`0.07 + i * 0.06}s`)
- No new keyframes introduced

### Accessibility
- Badge pills carry the full text "Client {score}" — screen readable without additional aria labels
- `ClientBehaviorCard` section label "Client Risk" and the score together convey meaning without color dependency
- Signal rows: icon is `aria-hidden="true"`, label text is the accessible description
- Color is supplementary (badge border color), never the sole signal carrier
