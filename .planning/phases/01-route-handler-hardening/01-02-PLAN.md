Replace `app/auth/callback/route.ts` with the following. The only change is capturing the result of `exchangeCodeForSession` and redirecting to `/login?error=auth_failed` when an error occurs. `NextResponse.redirect` is correct here — this is a GET redirect handler, not a JSON API route. `NextResponse` is already imported.

```typescript
import { createServerSupabaseClient } from '@/lib/supabase/server'
import { NextResponse } from 'next/server'

export async function GET(request: Request) {
  const { searchParams, origin } = new URL(request.url)
  const code = searchParams.get('code')

  if (code) {
    const supabase = await createServerSupabaseClient()
    const { error } = await supabase.auth.exchangeCodeForSession(code)
    if (error) {
      return NextResponse.redirect(`${origin}/login?error=auth_failed`)
    }
  }

  return NextResponse.redirect(`${origin}/dashboard`)
}
```

Conventions: no semicolons, single quotes, no trailing semicolons.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OAuth provider → callback route | Supabase returns error on expired/replayed code |
| URL query param → login page | ?error=auth_failed is attacker-controllable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ?error query param on login page | mitigate | Login page only reads the `error` param and checks `=== 'auth_failed'` — no param value is executed or reflected as HTML; any other value simply shows no banner |
| T-02-02 | Spoofing | OAuth code replay | mitigate | `exchangeCodeForSession` error is now detected and redirected — user sees clear failure message instead of landing unauthenticated on dashboard |
| T-02-03 | Information Disclosure | auth error message | accept | "Sign-in link expired" reveals that the code was valid but expired — acceptable; no secret information disclosed |
| T-02-04 | Tampering | DEFENSE_SYSTEM_PROMPT injection | accept | The off-topic guard is a prompt instruction, not a security control; sophisticated prompt injection can still override it; this is an acceptable risk for v1 |
</threat_model>
