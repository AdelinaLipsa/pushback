Create `lib/risk/index.ts` exporting the public `computeRisk` orchestrator plus re-exports for ergonomic consumer imports.

Signature: `export function computeRisk(project: Project, today?: Date | string): RiskResult`. Defaulting `today` lets the dashboard pass a single anchor for all projects (D-25 — same anchor → reproducible top-3 ordering across the request).

Implementation steps:

1. **Build RiskInput** — extract from `project`:
   - `projectId`: `project.id`
   - `today`: ISO string from the param (default: `new Date().toISOString()`). Cast to ISO once; never call `new Date()` again inside the orchestrator.
   - `paymentDueDate`: `project.payment_due_date`
   - `paymentReceivedAt`: `project.payment_received_at`
   - `projectValue`: `project.project_value`
   - `contractClauses`: derive from `project.contracts?.analysis`. ContractAnalysis today exposes `flagged_clauses[]`, `missing_protections[]`, `positive_notes[]`, `negotiation_priority[]` — none is a `clauses_present` array. Derive a presence list with this rule: scan `analysis.flagged_clauses[*].title` AND `analysis.positive_notes[*]` for token matches against the known clause vocabulary `['late_fee', 'kill_fee', 'payment_schedule', 'scope', 'revision_cap']`. A clause is "present" if any title or note string contains the token (case-insensitive, word-boundary). Absent analysis → `contractClauses = []` (forces all clause-gap signals to fire — correct default for "no contract on file").
   - `sentResponses`: `(project.defense_responses ?? []).filter(r => r.was_sent === true).map(r => ({ tool_type: r.tool_type, created_at: r.created_at }))`
   - `daysSinceLastResponse`: `null` if `sentResponses` is empty; otherwise floor((today_ms - max(created_at)) / 86400000).

2. **Score each dimension** — call `scorePayment`, `scoreScope`, `scoreChargeback` against the input. Capture the three `DimensionScore` results.

3. **Composite** — per D-18: `composite = clamp(round(0.4*payment.score + 0.3*scope.score + 0.3*chargeback.score), 0, 100)`. Round to integer (consumers render integers in badges per Phase 12 D-04 visual lock).

4. **Level** — `levelFromScore(composite)` from `lib/risk/weights.ts`.

5. **Top dimension** — pick the dimension with the largest `score`. Tie-break in declared order `payment > scope > chargeback` (deterministic).

6. **nextAction** — `pickNextAction(level, topDimension)`.

7. **topMitigation** — per D-20:
   - If `level === 'green'`: `topMitigation = null` (no actionable lever needed).
   - Otherwise: from the top dimension's `signals[]`, pick the signal with the highest `points` (ties broken by array order — first wins, since each scorer emits signals in deterministic order).
     - If the top dimension has zero signals (edge case: composite pulled up purely by other dimensions), fall back to the highest-scoring signal across all three dimensions.
   - `deltaPoints` = simulated reduction in **composite** if that signal were removed: subtract `signal.points` from the dimension score, recompute composite via the same weight formula, and report `composite - newComposite` rounded to integer.
   - `action`: `pickMitigationAction(signal.code)`. If the code is missing from `MITIGATION_TABLE`, set `topMitigation = null` (safer than emitting unhelpful text). Log nothing — pure function.

8. Return the locked `RiskResult` shape.

Re-export the public surface from one place so consumers write `import { computeRisk, RiskResult, RISK_LEVEL_COLORS } from '@/lib/risk'`:
- types: `RiskResult, RiskSignal, RiskLevel, RiskDimension, DimensionScore` (re-export from `./types`)
- constants: `RISK_LEVEL_COLORS, LEVEL_THRESHOLDS, COMPOSITE_WEIGHTS, levelFromScore` (re-export from `./weights`)

Performance: no async, no Promise, no `for await`. One pass per dimension. Per D-24 perf budget, the typical project (≤50 sent responses) must complete in <50ms.
