Adapter implementation:

```ts
import { computeRisk } from './risk'
// ...
export function computeClientRisk(project: ...): ClientRiskResult {
  const r = computeRisk(project)
  // Map Phase 14 level → Phase 12 level using the OLD 61 boundary, NOT 66:
  const level: ClientRiskLevel =
    r.composite >= 61 ? 'red' :
    r.composite >= 26 ? 'yellow' : 'green'
  // Collapse engine signals into Phase 12 signal shape (icon + label).
  // Engine signals carry { code, label, points, source } — Phase 12 wants { icon, label }.
  // Map source → icon using a small static map:
  //   'projects' → 'CreditCard'   (payment-domain signals)
  //   'responses' → 'Layers'      (response-driven signals)
  //   'contracts' → 'FileText'    (clause-gap signals)
  // For the chargeback-specific code, override with 'ShieldAlert'.
  // Drop signals with points <= 0 (the on-time bonus) — Phase 12 never surfaced negative signals.
  const SOURCE_ICON: Record<RiskSignal['source'], string> =
    { projects: 'CreditCard', responses: 'Layers', contracts: 'FileText' }
  const allSignals = [
    ...r.dimensions.payment.signals,
    ...r.dimensions.scope.signals,
    ...r.dimensions.chargeback.signals,
  ].filter(s => s.points > 0)
  const signals: ClientRiskSignal[] = allSignals.map(s => ({
    icon: s.code.startsWith('chargeback') || s.code.startsWith('dispute') || s.code.startsWith('review')
      ? 'ShieldAlert'
      : SOURCE_ICON[s.source],
    label: s.label,
  }))
  return { score: r.composite, level, signals }
}
```

Note: the adapter's level mapping (61, not 66) is intentional. Phase 12 consumers like the old `severity: 'client-risk'` row (now removed from the dashboard in Plan 03) and any external import we don't know about must see the original red boundary. Plan 03 already migrated the in-repo UI consumers to `@/lib/risk` directly, so this shim is purely a safety net.

Keep `RISK_WEIGHTS` and `LEVEL_LABELS` as plain constants (no re-export from the engine). Plan 04 makes no behavioral promise about `RISK_WEIGHTS` matching engine weights — it's frozen at its Phase 12 values for backward compatibility.
