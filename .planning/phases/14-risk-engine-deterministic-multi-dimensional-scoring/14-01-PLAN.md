Single payment reminders are not yet a risk signal — only repeated cadence pressure.

`daysLate` calculation:
- If `paymentDueDate === null`: zero contribution from late signals; only contract-clause signals apply.
- If `paymentReceivedAt !== null`: `daysLate = floor((received - due) / 86400000)`.
- If `paymentReceivedAt === null`: `daysLate = floor((today - due) / 86400000)` (unpaid + past-due).
- Negative `daysLate` = on-time (paid before due).

Each emitted signal carries the exact integer points it contributed. The signal label is a short human-readable English sentence (e.g. "Payment 23 days late", "Contract has no late-fee clause", "Repeated payment-cadence reminders sent"). Label phrasing is at Claude's discretion (CONTEXT D-19/D-20 lock the action sentences; signal labels are free).

After emitting signals, sum their points into `score`, then clamp per D-11: `score = Math.max(0, Math.min(100, score))`. The on-time bonus can make the running sum negative; floor before returning.

Return `{ score, signals }`. No mutation of `input`. No async. No console.
